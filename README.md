# American Roulette Project README

## 🏁 Project Overview
This project simulates a fully functional, multi-player American Roulette table. It is built with a microservices-oriented architecture, allowing for scalable deployment using Docker and Kubernetes.

## ⚙️ Technology Stack
*   **Frontend:** React (Client-Side UI)
*   **Backend:** Django REST Framework (Python)
*   **Database:** PostgreSQL
*   **Architecture:** Docker Compose / Kubernetes
*   **Communication:** WebSockets (for real-time state sync)

## 🚀 Getting Started (Local Development)

### Prerequisites
1.  Python 3.11+
2.  Node.js (for React)
3.  Docker & Docker Compose

### Setup Steps
1.  **Navigate to the root directory:**
    ```bash
    cd /path/to/roulette
    ```
2.  **Database Setup:** Initialize the PostgreSQL container and run initial migrations.
    ```bash
    docker-compose up -d db
    # Inside the container, run the migrations:
    docker exec -it roulette_db python manage.py makemigrations roulette_api
    docker exec -it roulette_db python manage.py migrate
    ```
3.  **Backend Setup:** Install dependencies and run the Django server.
    ```bash
    # Install Python dependencies first
    pip install -r backend/requirements.txt
    # Run the server (Requires setting up the .env file)
    export SECRET_KEY='your-secret-key'
    django-admin startstartproject roulette_api
    python manage.py runserver 0.0.0.0:8000
    ```
4.  **Frontend Setup:** Install Node dependencies and start the React development server.
    ```bash
    cd frontend
    npm install
    npm start
    ```

## 🧪 Running the Simulation
1.  **Live Game:** Access the React app on `http://localhost:3000`. Players can place bets and initiate the spin.
2.  **Strategy Testing:** To test a strategy, access the dedicated simulation endpoint (requires API modification) and send a POST request to `/api/v1/strategies/run_simulation/` with the strategy details.
3.  **AI Agent Play:** To test agents, use the `/api/v1/agents/agent_bet/` endpoint, calling it multiple times sequentially to simulate agent turns.

## 📝 Key Notes
*   **Betting Logic:** The core math is centralized in `backend/roulette_api/views.py` and needs thorough unit testing.
*   **Concurrency:** All transactions *must* run within a database transaction block (`@transaction.atomic`) to prevent race conditions.
